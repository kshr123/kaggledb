# 9. タグ管理仕様

## 9.1 Phase 1（固定タグ）

**実装方針**
- 初期データとして固定タグをDB投入
- UIでは編集不可（表示のみ）
- APIは取得のみ（GET /api/tags）

**将来の拡張準備**
- テーブル設計は編集可能な構造
- `is_system` フラグで初期タグを保護
- APIエンドポイントは定義済み（実装は未実装）

## 9.2 Phase 2以降（UI編集可能）

**追加機能**
```
設定画面（将来実装）
├─ タグ管理
│  ├─ タグ一覧（カテゴリ別表示）
│  ├─ [+ 新規タグ追加] ボタン
│  ├─ 編集（名前、カテゴリ変更）
│  └─ 削除（使用中は警告、システムタグは削除不可）
```

**実装するAPI（将来）**
- `POST /api/tags` - タグ追加
- `PUT /api/tags/{id}` - タグ編集
- `DELETE /api/tags/{id}` - タグ削除（is_system=false のみ）
