# 8. 非機能要件

## 8.1 パフォーマンス
- **初回ページロード**: 3秒以内
- **ページ遷移**: 1秒以内（SPAの利点）
- **検索・フィルタ**: 500ms以内
- **API レスポンス**: 1秒以内
- **LLM処理**: 1件あたり5秒以内（バックグラウンド処理）

## 8.2 セキュリティ
- API Keyは環境変数で管理（`.env` ファイル）
- `.env` ファイルは Git 管理外（`.gitignore` に追加）
- SQLインジェクション対策（パラメータ化クエリ）
- CORS設定（Next.jsからのアクセスのみ許可）
- 本番環境では認証機能追加を検討

## 8.3 拡張性
- Phase 2, 3の機能追加を想定した設計
- DB構造の変更に対応しやすい設計
- コンポーネントベースの開発（再利用性）
- APIのバージョニング（将来的に）

## 8.4 保守性
- コードのモジュール化
- 明確な責務分離（フロント/バック）
- TypeScriptによる型安全性
- ログ出力（エラー追跡）
- コメント・ドキュメント

## 8.5 開発体験
- ホットリロード（Next.js, FastAPI両方）
- TypeScriptの型補完
- ESLint + Prettier でコード品質維持
- Docker Composeで環境統一
