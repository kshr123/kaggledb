# 1. システム概要

## 1.1 目的
Kaggleのコンペ情報、ディスカッション、上位解法を体系的に管理・検索できるナレッジベースシステムの構築。

## 1.2 ターゲットユーザー
- 自分自身（個人利用）
- 将来的には公開も検討

## 1.3 利用環境
- ローカル開発環境（Docker Compose）
- フロントエンド: http://localhost:3000
- バックエンドAPI: http://localhost:8000
- データはローカルに保存（SQLite）

## 1.4 アーキテクチャ概要

```
┌─────────────────────────────────────────┐
│ Next.js (TypeScript)                    │
│ - UI/UX                                 │
│ - ルーティング                           │
│ - 状態管理                               │
│ - クライアントサイド処理                  │
│ Port: 3000                              │
└─────────────────────────────────────────┘
              ↓ REST API (JSON)
┌─────────────────────────────────────────┐
│ FastAPI (Python 3.13)                   │
│ - REST API                              │
│ - Kaggle API連携                        │
│ - OpenAI API連携（LLM処理）              │
│ - SQLite操作                            │
│ - バッチ処理                             │
│ - Playwright（スクレイピング）           │
│ Port: 8000                              │
└─────────────────────────────────────────┘
              ↓
┌─────────────────────────────────────────┐
│ SQLite Database                         │
│ - competitions                          │
│ - discussions                           │
│ - solutions                             │
│ - tags                                  │
└─────────────────────────────────────────┘
```
